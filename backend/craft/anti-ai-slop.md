# AI 刻板模式避免规则

以下规则区分"有经验开发者作品"和"AI 默认输出"。生成代码时必须遵守。

## P0 必须避免

1. **Tailwind indigo/purple 默认色系** — 不要使用 `#6366f1`、`#8b5cf6`、`#a855f7` 作为主色调。选择有明确语义的设计色系（如 blue、emerald、rose）。Indigo 是 AI 生成的标志性默认色。

2. **紫蓝渐变 Hero** — 不要使用 `purple→blue`、`blue→cyan` 双色渐变作为 Hero 背景。纯色配好排版胜过万能渐变。

3. **Emoji 作为功能图标** — 不要在标题、按钮、列表中大量使用 `✨🚀🎯⚡🔥💡` 等 emoji 替代功能图标。使用 SVG 图标或纯文字描述。

4. **AI 仪表盘卡片模板** — 避免"大圆角卡片 + 彩色左边框"的组合，这是最典型的 AI 仪表盘样式。

5. **伪造的数据指标** — 不要生成"10× 更快"、"99.9% 可用性"等无现实依据的指标数据。使用真实或标注为示例的数据。

6. **Lorem ipsum 占位文本** — 不使用 lorem ipsum、功能一/二/三等无意义占位。用具体的、有意义的中文描述。

## P1 应当避免

- **标准页面骨架无变化**（Hero → 功能 → 定价 → FAQ → CTA）：至少引入一个有变化的区块（如用户评价墙、内嵌产品演示）
- **超过 12 个裸 hex 颜色值**：应使用 CSS 变量或 Tailwind 语义色
- **过度圆角风**：`rounded-3xl` 或 `rounded-full` 滥用，卡片圆角控制在 `rounded-lg`/`rounded-xl`

## 如何增加设计感

- **80% 常规模式 + 20% 特色选择**：有一个大胆的字体选择、一处意外的间距比例、一个让人记住的微交互
- 按钮文案用"开始记录"比"开始使用"更有温度
